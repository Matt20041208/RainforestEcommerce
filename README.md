# 🌿 雨林电商系统 - 项目设计方案总结

## 一、系统架构设计

### 1.1 架构模式
采用 **前后端分离** 的 B2C 电商架构：
- **前端**：Vue 3 SPA 单页应用
- **后端**：Spring Boot RESTful API 服务
- **数据库**：MySQL（持久化）+ Redis（缓存）

### 1.2 分层设计
```
┌─────────────────────────────────────┐
│         前端展示层                   │
│    (Vue 3 + Vue Router + Vite)      │
└──────────────┬──────────────────────┘
               │ HTTP/HTTPS
┌──────────────▼──────────────────────┐
│         网关/安全层                   │
│   (Spring Security + JWT Filter)    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         控制层                       │
│   (RESTful API Controller)          │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         业务服务层                   │
│      (Service + Business Logic)     │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         数据访问层                   │
│      (MyBatis Mapper)               │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         数据存储层                   │
│   (MySQL + Redis)                   │
└─────────────────────────────────────┘
```

## 三、核心模块设计

### 3.1 用户模块

**设计要点**：
- **双角色系统**：支持 `user`（普通用户）和 `seller`（商家）两种角色
- **密码安全**：使用 BCrypt 加密存储
- **JWT 认证**：无状态 Token 认证机制

**核心功能**：
- 用户注册（用户名唯一性校验）
- 用户登录（角色区分）
- 个人信息管理
- 头像上传

### 3.2 商品模块

**设计要点**：
- **商品分类**：支持多级分类，启用/禁用控制
- **商品状态**：enabled 字段控制商品上架/下架
- **图片存储**：本地文件存储，UUID 命名

**核心功能**：
- 商品 CRUD（商家权限）
- 商品列表展示（分页）
- 商品搜索
- 商品详情查看

### 3.3 购物车模块

**设计要点**：
- **Redis 存储**：使用 Redis Hash 存储购物车数据
- **数据结构**：`cart:{userId}` → List<CartItem>
- **实时计算**：添加/修改商品时自动计算总价

**核心功能**：
- 添加商品到购物车
- 修改商品数量
- 移除商品
- 清空购物车
- 查看购物车

### 3.4 订单模块

**设计要点**：
- **订单类型**：区分用户订单（购买）和商家订单（采购）
- **订单状态流转**：待付款 → 待发货 → 待收货 → 已完成
- **订单项关联**：订单与商品多对多关系

**核心功能**：
- 创建订单（购物车 → 订单）
- 查看订单列表
- 更新订单状态
- 订单详情查看

### 3.5 评论模块

**设计要点**：
- **评分系统**：1-5 星评分
- **双向关联**：评论关联用户和商品
- **商家回复**：商家可回复用户评论

**核心功能**：
- 发表评论（购买后）
- 查看商品评论
- 商家回复评论

### 3.6 地址模块

**设计要点**：
- **地址结构**：省/市/区/详细地址
- **默认地址**：支持设置默认收货地址

**核心功能**：
- 添加收货地址
- 修改/删除地址
- 设置默认地址

## 五、安全设计

### 5.1 认证机制

**JWT Token 流程**：
```
1. 用户登录 → 验证用户名密码
2. 生成 JWT Token（包含 username, role, 过期时间）
3. 返回 Token 给客户端
4. 客户端存储 Token（localStorage）
5. 每次请求在 Header 携带：Authorization: Bearer {token}
6. 服务端通过 JwtAuthenticationFilter 验证 Token
```

### 5.2 授权规则

| 接口类型 | 认证要求 | 说明 |
|---------|---------|------|
| 登录/注册 | ❌ 无需认证 | 公开接口 |
| 商品/分类 | ❌ 无需认证 | 公开浏览 |
| 静态资源 | ❌ 无需认证 | 图片等 |
| 购物车/订单 | ✅ 需要认证 | 需登录 |
| 用户信息 | ✅ 需要认证 | 需登录 |

### 5.3 密码安全
- 使用 BCrypt 加密算法
- 密码强度校验（前端 + 后端）
- Token 过期时间：24 小时

## 六、业务流程设计

### 6.1 用户购物流程

```
注册/登录
    ↓
浏览商品（按分类/搜索）
    ↓
查看商品详情
    ↓
加入购物车
    ↓
管理购物车（增删改）
    ↓
确认订单（选择地址）
    ↓

    ↓

    ↓
查看订单状态
    ↓
确认收货
    ↓

```

### 6.2 商家销售流程

```
商家注册/登录
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓
```

## 七、关键技术实现

### 7.1 Redis 购物车实现

**数据结构**：
```
Key: cart:{userId}
Value: JSON Array of CartItem
```

**优势**：
- 读写速度快
- 支持过期时间
- 减少数据库压力

### 7.2 文件上传实现

**流程**：
1. 前端上传 MultipartFile
2. 后端生成 UUID 文件名
3. 保存到指定目录
4. 返回访问 URL

**配置**：
```properties
file.upload.path=./uploads
file.upload.url=/uploads
spring.servlet.multipart.max-file-size=10MB
```

### 7.3 分页查询实现

使用 PageHelper 插件：
```java
PageHelper.startPage(pageNum, pageSize);
List<Product> products = productMapper.selectAll();
PageInfo<Product> pageInfo = new PageInfo<>(products);
```

## 九、项目亮点

1. **双角色系统**：同时支持用户和商家，业务闭环
2. **前后端分离**：架构清晰，易于维护和扩展
3. **JWT 认证**：无状态认证，适合分布式部署
4. **Redis 购物车**：高性能缓存，提升用户体验
5. **RESTful API**：接口规范，易于对接第三方
6. **代码规范**：分层清晰，遵循 MVC 模式

---

## 十、总结

本项目是一个功能完整的 B2C 电商平台，采用主流的 Spring Boot + Vue 3 技术栈，实现了用户购物和商家销售的核心业务流程。系统架构清晰，模块划分合理，具有良好的扩展性和可维护性。通过 JWT 认证、Redis 缓存等技术手段，保证了系统的安全性和性能。
